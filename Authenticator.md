A "thing" (or "things" when using more than one) used by an [[Entity]], along with an [[Identifier]] to [[Authenticate]] themself (for a [[Person Entity]]) / itself (for a [[Non-person Entity]]).

Authenticators may be described as falling into three types - something you:
- **Have**: AKA possession factors
- **Know**: AKA knowledge factors
- **Are**: AKA biometric factors

Using two authenticators of different type in the same authentication process is referred to as [[Multi-factor Authentication]]

Each type of factor has various different implementations - e.g. a USB security key (possession), passphrase (knowledge), or fingerprint (biometric).

The [[Canadian Centre for Cyber Security]] publishes guidance on the management and use of authenticators in [[ITSP.30.031]] (though, authenticators are referred to as "tokens" in version 3.  Future versions will use "authenticator" in alignment with [[NIST]] publications.)
