The time at which events take place in the practice of Identity and Access Management is a critically important concept, as much of IAM can be considered as automation and/or orchestration of process.

These times are named relative to the [[Session]] when an access event occurs - i.e. the time during which an entity attempts to perform an action on a resource.

The three time periods of note are:
- [[Admin-time]]: performed before an access event
- [[Real-time]]: performed to establish a session, and during the session in which an access event occurs
- [[Post event-time]]: performed after the session in which an access event occurs is terminated
