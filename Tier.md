May refer to:
- the number of levels of [[Certification Authorities]] in a [[PKI Hierarchy]] - e.g. a "two-tier" PKI has a [[Root Certification Authority]] and one or more [[Issuing Certification Authority]] which are directly subordinate to the Root.

- layers in an IT infrastructure security model for separating duties and access based on sensitivity of assets - e.g. tier 0 = foundational infrastructure where its compromise could affect all other layers (e.g. [[IAM]] and its directly supporting infrastructure, since it controls access at all other levels); tier 1 = critical infrastructure whose compromise would not affect tier-0, but could affect other tiers (e.g. network and hosting infrastructure, besides that directly supporting tier-0)

- hierarchical levels in an IT support structure - e.g. tier 1 = generic help desk support, tier 2 = specialized support, tier 3 = expert support
